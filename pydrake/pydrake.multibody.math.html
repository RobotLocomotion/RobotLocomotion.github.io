This class represents a spatial acceleration A and has 6 elements with an angular (rotational) acceleration Œ± (3-element vector) on top of a translational (linear) acceleration ùêö (3-element vector). Spatial acceleration represents the rotational and translational acceleration of a frame B with respect to a measured-in frame M. This class assumes that both the angular acceleration Œ± and translational acceleration ùêö are expressed in the same expressed-in frame E. class pydrake.multibody.math.SpatialVelocity_[float]¬∂

This class represents a spatial velocity V (also called a twist) and has 6 elements with an angular (rotational) velocity œâ (3-element vector) on top of a translational (linear) velocity v (3-element vector). Spatial velocity represents the rotational and translational motion of a frame B with respect to a measured-in frame M. This class assumes that both the angular velocity œâ and translational velocity v are expressed in the same expressed-in frame E. This class only stores 6 elements (namely œâ and v) and does not store the underlying frames B, M, E. The user is responsible for explicitly tracking the underlying frames with multibody_quantities ‚Äúmonogram notation‚Äù. For example, V_MB_E denotes frame B‚Äôs spatial velocity measured in frame M, expressed in frame E and contains œâ_MB_E (B‚Äôs angular velocity measured in M, expressed in E) and v_MBo_E (Bo‚Äôs translational velocity measured in M, expressed in E), where Bo is frame B‚Äôs origin point. For an multibody_frames_and_bodies ‚Äúoffset frame‚Äù Bp, the monogram notation V_MBp_E denotes the spatial velocity of frame Bp measured in M, expressed in E. Details on spatial vectors and monogram notation are in sections multibody_spatial_vectors and multibody_quantities.
