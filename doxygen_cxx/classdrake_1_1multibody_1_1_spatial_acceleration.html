SpatialAcceleration< T > Class Template Reference

Detailed Description

template<typename T>
class drake::multibody::SpatialAcceleration< T >

This class represents a spatial acceleration A and has 6 elements with an angular (rotational) acceleration Œ± (3-element vector) on top of a translational (linear) acceleration ùêö (3-element vector).

Spatial acceleration represents the rotational and translational acceleration of a frame B with respect to a measured-in frame M. This class assumes that both the angular acceleration Œ± and translational acceleration ùêö are expressed in the same expressed-in frame E. This class only stores 6 elements (namely Œ± and ùêö) and does not store the underlying frames B, M, E. The user is responsible for explicitly tracking the underlying frames with monogram notation. For example, A_MB_E denotes frame B's spatial acceleration measured in frame M, expressed in frame E and contains alpha_MB_E (B's angular acceleration measured in M, expressed in E) and a_MBo_E (Bo's translational acceleration measured in M, expressed in E), where Bo is frame B's origin point. For an offset frame Bp, the monogram notation A_MBp_E denotes frame Bp's spatial acceleration measured in M, expressed in E. Details on spatial vectors and monogram notation are in sections Spatial Vectors and Multibody Quantities.

The typeset for A_MB is \(\,{^MA^B}\) and its definition is \(^MA^B = \frac{^Md}{dt}\,{^MV^B}\,\), where \({^MV^B}\) is frame B's spatial velocity in frame M and \(\frac{^Md}{dt}\) denotes the time derivative taken in frame M. To differentiate a vector, we need to specify in what frame the time derivative is taken, see [Mitiguy 2022, ¬ß7.2] for an in-depth discussion. Time derivatives in different frames are related by the "Transport Theorem", which in Drake is implemented in drake::math::ConvertTimeDerivativeToOtherFrame(). In source code (monogram) notation, we write A_MB = DtM(V_MB), where DtM() denotes the time derivative in frame M. Details on vector differentiation is in section Time Derivatives of Multibody Quantities.

[Mitiguy 2022] Mitiguy, P., 2022. Advanced Dynamics & Motion Simulation.

The returned spatial acceleration A_MC_E contains an angular acceleration Œ±_MC_E and translational acceleration a_MCo_E that are calculated as: 
 Œ±_MC_E  = Œ±_MB_E + Œ±_BC_E + œâ_MB_E x œâ_BC_E
 a_MCo_E = a_BCo_E + Œ±_MB_E x p_BoCo_E + œâ_MB_E x (œâ_MB_E x p_BoCo_E)
         + 2 œâ_MB_E x v_BCo_E + a_BCo_E
If frame C is rigidly fixed to frame B, A_BC_E = 0 and V_BC_E = 0 and this method produces a Shift() operation (albeit inefficiently). The previous equations show composing spatial acceleration is not simply adding A_MB + A_BC and these equations differ significantly from their spatial velocity counterparts. For example, angular velocities simply add as 
  œâ_MC = œâ_MB + œâ_BC,   but 3D angular acceleration is more complicated as
  Œ±_MC = Œ±_MB + Œ±_BC + œâ_MB x œâ_BC

Derivation

Rotational acceleration component

œâ_MC (frame C's angular velocity in frame M) can be calculated with the angular velocity addition theorem as 

  œâ_MC = œâ_MB + œâ_BC

Œ±_MC (frame C's angular acceleration measured in frame M) is defined as the time-derivative in frame M of œâ_MC, and can be calculated using the "Transport Theorem" (Golden rule for vector differentation) which converts the time-derivative of a vector in frame M to frame B, e.g., as DtM(œâ_BC) = DtB(œâ_BC) + œâ_MB x œâ_BC, as 

  Œ±_MC = DtM(œâ_MC) = DtM(œâ_MB) + DtM(œâ_BC)
                   =     Œ±_MB  + DtB(œâ_BC) + œâ_MB x œâ_BC
                   =     Œ±_MB  +     Œ±_BC  + œâ_MB x œâ_BC   (End of proof).

Translational acceleration component

v_MCo (frame C's translational velocity in frame M) is calculated in SpatialVelocity::ComposeWithMovingFrameVelocity) as 

  v_MCo = v_MBo + œâ_MB x p_BoCo + v_BCo

a_MCo (frame C's translational acceleration measured in frame M) is defined as the time-derivative in frame M of v_MCo, calculated as 

 a_MCo = DtM(v_MCo)                             Definition.
       = DtM(v_MBo + œâ_MB x p_BoCo + v_BCo)     Substitution.
       = DtM(v_MBo) + DtM(œâ_MB) x p_BoCo + œâ_MB x DtM(p_BoCo) + DtM(v_BCo)
       =     a_MBo  +     Œ±_MB  x p_BoCo + œâ_MB x DtM(p_BoCo) + DtM(v_BCo)

The last two terms are modified using the "Transport Theorem" (Golden rule for vector differentation) which converts time-derivatives of vectors in frame M to frame B via DtM(vec) = DtB(vec) + œâ_MB x vec. 

 DtM(p_BoCo) = DtB(p_BoCo) + œâ_MB x p_BoCo
             =     v_BCo   + œâ_MB x p_BoCo
 DtM(v_BCo)  = DtB(v_BCo)  + œâ_MB x v_BCo
             =     a_BCo   + œâ_MB x v_BCo

Combining the last few equations proves the formula for a_MCo as: 

  a_MCo = a_MBo + Œ±_MB x p_BoCo + œâ_MB x (œâ_MB x p_BoCo)
        + 2 œâ_MB x v_BCo + a_BCo                           (End of proof).

Some terms in the previous equation have names, e.g., 

  centripetal acceleration   œâ_MB x (œâ_MB x p_BoCo)
  Coriolis acceleration    2 œâ_MB x v_BCo
  Coincident point acceleration, i.e., acceleration of the point of frame
  B coincident with Co      a_MBo + Œ±_MB x p_BoCo + œâ_MB x (œâ_MB x p_BoCo)

Note: The coincident point acceleration can be calculated with a Shift().

Note: The three cross products appearing in the previous calculation of a_MCo can be reduced to one, possibly improving efficiency via 

  œâ_MB x (œâ_MB x p_BoCo) + 2 œâ_MB x v_BCo = œâ_MB x (v_MCo - v_MBo + v_BCo)

To show this, we rearrange and substitute our expression for v_MCo. 

          v_MCo = v_MBo + œâ_MB x p_BoCo + v_BCo        which rearranges to
  œâ_MB x p_BoCo = v_MCo - v_MBo - v_BCo.             Substitution produces
  œâ_MB x (œâ_MB x p_BoCo) = œâ_MB x (v_MCo - v_MBo - v_BCo)           Hence,
  œâ_MB x (œâ_MB x p_BoCo) + 2 œâ_MB x v_BCo = œâ_MB x (v_MCo - v_MBo + v_BCo)

In-place shift of a SpatialAcceleration from a frame B to a frame C, where both B and C are fixed to the same frame or rigid body.

On entry, this is A_MB_E (frame B's spatial acceleration measured in a frame M and expressed in a frame E). On return this is modified to A_MC_E (frame C's spatial acceleration measured in frame M and expressed in frame E). The components of A_MC_E are calculated as: 

  Œ±_MC_E = Œ±_MB_E           (angular acceleration of `this` is unchanged).
 a_MCo_E = a_MBo_E + Œ±_MB_E x p_BoCo_E + œâ_MB_E x (œâ_MB_E x p_BoCo_E)
Parameters
[in]offsetwhich is the position vector p_BoCo_E from Bo (frame B's origin) to Co (frame C's origin), expressed in frame E. p_BoCo_E must have the same expressed-in frame E as this spatial acceleration.
[in]angular_velocity_of_this_framewhich is œâ_MB_E, frame B's angular velocity measured in frame W and expressed in frame E.
See also
Shift() to shift spatial acceleration without modifying this. Use ComposeWithMovingFrameAcceleration() if frame C is moving on frame B.

Derivation

Rotational acceleration component

Frame B and frame C are fixed (e.g., welded) to the same rigid object. Hence frames B and C always rotate together at the same rate and: 

  œâ_MC_E = œâ_MB_E
  Œ±_MC_E = Œ±_MB_E

Translational acceleration component

Since frames B and C are fixed to the same rigid object, the translational velocity of Co (frame C's origin) measured in frame M can be calculated as 

  v_MCo = v_MBo + œâ_MB x p_BoCo

Point Co's translational acceleration in frame M is: 

  a_MCo = DtM(v_MCo)                      (definition of acceleration).
        = DtM(v_MBo  + œâ_MB x p_BoCo)     (substitution)
        = DtM(v_MBo) + DtM(œâ_MB) x p_BoCo + œâ_MB x DtM(p_BoCo)
        =     a_MBo  +     Œ±_MB  x p_BoCo + œâ_MB x DtM(p_BoCo)

The "Transport Theorem" converts the time-derivative of the last term from DtM() to DtB() – see math::ConvertTimeDerivativeToOtherFrame(), as 

  DtM(p_BoCo) = DtB(p_BoCo) + œâ_MB x p_BoCo
              =          0  + œâ_MB x p_BoCo
