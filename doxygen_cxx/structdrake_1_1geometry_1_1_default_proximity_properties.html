DefaultProximityProperties Struct Reference

Detailed Description

These properties will be used as defaults when the geometry as added via API calls or parsed from model files doesn't say anything more specific.

See also
Hydroelastic Contact User Guide, Hydroelastic Quick-start, Properties for hydroelastic contact, Continuous Approximation of Coulomb Friction.

Member Data Documentation

◆ compliance_type

std::string compliance_type {"undefined"}

There are three valid options for compliance_type:

  • "undefined": hydroelastic contact will not be used.
  • "rigid": the default hydroelastic compliance type will be rigid; note that rigid-rigid contact is not supported by the hydroelastic contact model, but is supported by point contact (multibody::ContactModel::kPoint or multibody::ContactModel::kHydroelasticWithFallback).
  • "compliant": the default hydroelastic compliance type will be compliant.

◆ dynamic_friction

std::optional<double> dynamic_friction {0.5}
